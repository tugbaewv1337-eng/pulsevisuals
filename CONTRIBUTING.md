# Направления по Контрибьюции

Спасибо за их интерес к содействию развитию PulseVisuals!

## Как начать

### 1. Fork репозитория
```bash
git clone https://github.com/YOUR_USERNAME/PulseVisuals.git
cd PulseVisuals
```

### 2. Настройка девелопмент окружения

#### Предварительные компоненты
- Java 21 SDK
- Git
- IntelliJ IDEA (or your IDE)
- Gradle (comes with wrapper)

#### Открытие проекта в IDE

```bash
# Генерация run configurations
./gradlew genSources

# Открытие в IntelliJ IDEA
# Вберите File -> Open
# Навигируйте к build.gradle
# Нажмите Open as Project
```

### 3. Создание ветки

```bash
git checkout -b feature/your-feature-name
```

Обычно именуются ветки:
- `feature/description` - Новые фичи
- `bugfix/description` - Отклонение ошибок
- `docs/description` - Обновления документации

## Код квалификации и рекомендации

### Кодовые стандарты

- Названия в camelCase для методов и вариаблей
- Названия в PascalCase для классов
- UPPER_CASE для статичных финальных постоянных

### Написание новых наружных классов

Ознакомьтесь с структурой и существующими классами:
- Рендеры в `client.visual.renderers`
- Утилиты в `client.util`
- Миксины в `mixin`

### Коментарии и документация

```java
/**
 * Описание метода.
 *
 * @param param1 Описание param1
 * @return Описание возвращаемого значения
 */
public void methodName(String param1) {
    // Конкретные комментарии к сложным операциям
}
```

## Цикл разработки

### 1. Написание кода
```bash
# Можете часто пересобирать для проверки
./gradlew build

# Как повторно загружать в Minecraft
# Запустите с Gradle -> Tasks -> fabric -> runClient
```

### 2. Тестирование
- Тестируйте регулярно во время разработки
- Тестируйте оба сингле и мултиплейер режимы
- Проверьте с несколькими типами оружия

### 3. Коммит

Говорите о желтелю ниже добавить превосходные коммит сообщения:

```bash
git add .
git commit -m "feat: add new particle effect"
# ор матика
# "fix: correct trajectory calculation"
# "docs: update README"
# "refactor: improve code structure"
```

#### Коммит Типы:
- **feat**: Новая фича
- **fix**: Отклонение бага
- **docs**: Изменения документации
- **refactor**: Переструктурирование кода
- **test**: Добавление тестов
- **chore**: Обновление вымененных или зависимостей

### 4. Пуш и Pull Request

```bash
# Укажите свож пусх
# используя ставки пройти с мест видяия
git push -u origin feature/your-feature-name
```

## Pull Request Процесс

### 1. Открытие Pull Request
- Однотипные PR для каждых отдельных фич как багфикс
- Осияйте с достаточным описанием

### 2. PR Шаблон

```markdown
## Описание
Окончательное описание ваших изменений.

## Тип изменений
- [ ] New feature
- [ ] Bug fix
- [ ] Breaking change
- [ ] Documentation update

## Проверно НаН
- [ ] Отвечает моду кодовым стандартам
- [ ] Применил документацию (когда необходимо)
- [ ] Оттестирован в Minecraft
- [ ] Не вводит критичные баги

## Мотивация
Описываюте мотивации и контекст тех изменений.
```

### 3. Ревиюаемые Наказы
- Программисты майнтейнера осмотрят все PR
- Обратная связь должна быть окончена до мержа
- Овчарина орыпки оперируются ан всеми PR незвисимо от размера

## Issues и Feature Requests

### Если вы нашли багу
1. Проверьте issues если баг не известна
2. Открытие Issue и творчет с этапами для всождения
3. Очку баг-репоорт вклюте несколькодоскриптсия

### Новая фича средреквесты
1. Проверьте issues если започа не предлагалась
2. Открытие Issue и до отличия
3. Ограс оды эджы дезайн

1. Добавьте раетвит брифёр в IntelliJ
2. Нажмите Debug в градле настройке
3. Начните runClient
